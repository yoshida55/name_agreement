# コーディング規約

## 基本フォーマット

- インデントはスペース2個
- コードはすべて小文字
- 長いテキストはコード内で改行せず、エディタの折り返し表示（Alt+Z）を使う

---

## 単位のルール

### 基本ルール

| 単位 | 使う場面 |
|------|----------|
| rem | サイズ指定（font-size, width, height, margin, padding 等） |
| em | line-height, letter-spacing のみ |
| px | 使わない |

### XDからの計算方法

※ reset.cssで `1rem = 10px` に設定されている前提

| プロパティ | 計算式 | 例 |
|------------|--------|-----|
| font-size等 | XDの値 ÷ 10 | 32px → 3.2rem |
| line-height | XDの値 ÷ フォントサイズ | 51px ÷ 34px → 1.5em |
| letter-spacing | XDの値 ÷ 1000 | 50 → 0.05em |

## HTMLのルール

### セマンティックHTMLを使う

「意味のあるタグ」を使うことで、検索エンジンや読み上げソフトが内容を正しく理解できる。

| 場所 | 使うタグ |
|------|----------|
| ヘッダー部分 | `<header>` |
| フッター部分 | `<footer>` |
| ナビゲーション | `<nav>` |
| メインコンテンツ | `<main>` |
| セクション | `<section>` |
| 見出し | `<h1>`〜`<h6>`（重要度順） |

```html
<!-- ❌ NG：全部divで書く -->
<div class="header">...</div>
<div class="footer">...</div>

<!-- ⭕ OK：意味のあるタグを使う -->
<header class="site_header">...</header>
<footer class="site_footer">...</footer>
```

### 見出しにはhタグを使う

「About」「Works」などのセクションタイトルは見出し。`<h1>`〜`<h6>`を使ってページの構造を明確にする。

```html
<h1>My Work</h1>           <!-- ページ全体のタイトル（1ページに1つ） -->
<h2>About</h2>             <!-- セクションの見出し -->
<h2>Works</h2>
<h3>Web Design</h3>        <!-- サブ見出し -->
```

### リンクは必ずaタグで作る

クリックできる要素（ナビメニュー、ボタンなど）は`<a>`タグを使う。

```html
<!-- ❌ NG：divやspanで作る -->
<div class="button">See More</div>

<!-- ⭕ OK：aタグで作る -->
<a href="works.html" class="button">See More</a>
```

### 外部リンク

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer"></a>
```

※ `rel="noopener noreferrer"` はセキュリティ対策（リンク先から元ページを操作されるのを防ぐ）

### 画像のalt属性

名詞または短い説明で書く。「〜です」のような文章は書かない。目が見えない人にも伝わる内容を簡潔に。

```html
<!-- ⭕ OK：名詞・短い説明 -->
<img src="photo.jpg" alt="宮地翔のプロフィール写真">
<img src="logo.png" alt="My Workのロゴ">
<img src="bike.jpg" alt="自転車">

<!-- ❌ NG：文章になっている -->
<img src="photo.jpg" alt="これは自転車の写真です">
<img src="logo.png" alt="こちらが私たちのロゴになります">
```

### 改行のルール

デザインに合わせて`<br>`を入れない。テキストはCSSのwidthで自然に折り返させる。

```html
<!-- ❌ NG：デザインに合わせて手動で改行 -->
<p>
  日々の料理レシピをまとめています。<br />
  和食や洋食、中華、お菓子まで<br />
  いろいろな料理レシピをアップしています。
</p>

<!-- ⭕ OK：自然に折り返す -->
<p>日々の料理レシピをまとめています。和食や洋食、中華、お菓子までいろいろな料理レシピをアップしています。</p>

<!-- ⭕ OK：SP専用改行が必要な場合のみ -->
<p>いろいろな料理<br class="sp_br">レシピをアップしています。</p>
```

### 段落（pタグ）の使い方

「見た目で改行したいから」という理由でpタグを分けない。ひとつの段落はひとつのpタグ。

```html
<!-- ❌ NG：見た目の改行のためにpタグを分けている -->
<p>日々の料理レシピをまとめています。</p>
<p>和食や洋食、中華、お菓子まで</p>
<p>いろいろな料理レシピをアップしています。</p>

<!-- ⭕ OK：ひとつの段落はひとつのpタグ -->
<p>
  日々の料理レシピをまとめています。
  和食や洋食、中華、お菓子までいろいろな料理レシピをアップしています。
</p>
```

### セクションにはコメントを残す

HTMLを後から見たとき「ここ何のセクションだっけ？」とならないように、コメントで目印を残す。

```html
<!-- ❌ NG：コメントなし -->
<section>
  <h1>Recipe Diary</h1>
  <p>日々の料理レシピをまとめています。</p>
</section>

<!-- ⭕ OK：セクションごとに役割を書いておく -->
<!-- ===============================================
 サイト紹介
=============================================== -->
<section class="intro">
  <h1>Recipe Diary</h1>
  <p>日々の料理レシピをまとめています。</p>
</section>
```

### リスト・テーブルは改行してインデントする

入れ子構造は「親子関係」が見えるように書く。

```html
<!-- ⭕ OK：改行とインデントで親子関係が見える -->
<ul>
  <li>Instagram</li>
  <li>Twitter</li>
  <li>Facebook</li>
</ul>
```

## CSSのルール

### セレクタ

- 子孫セレクタは1段階まで
- タグを直接指定しない

```css
/* ⭕ OK：1段階（親 → 子） */
.header .logo {
}
.nav_list .nav_link {
}

/* ❌ NG：2段階以上（親 → 子 → 孫） */
.header .nav .nav_link {
}
.recipe_list .recipe_card .recipe_card_image {
}
```

### position: absolute は最終手段

Flexboxで実現できるレイアウトには使わない。重ねる・特定位置への配置のみ使用する。

```css
/* ❌ NG：Flexboxで実現できるものにabsoluteを使う */
/* ヘッダーのロゴとナビを横並びにしたい */
.site_header_logo {
  position: absolute;
  left: 40px;
}
.global_nav {
  position: absolute;
  right: 40px;
}

/* ⭕ OK：Flexboxを使う */
.site_header_row {
  display: flex;
  justify-content: space-between;
}

/* ⭕ OK：画像の上にテキストを重ねる（absoluteの正しい使い方） */
.card {
  position: relative;
}
.card_badge {
  position: absolute;
  top: 0.5rem;
  right: 0.5rem;
}
```

### margin・paddingはまとめて書く

ショートハンド（省略記法）を使うとコードがスッキリする。

| 書き方 | 意味 |
|--------|------|
| `padding: 10px;` | 上下左右すべて10px |
| `padding: 10px 20px;` | 上下10px、左右20px |
| `padding: 10px 20px 30px;` | 上10px、左右20px、下30px |
| `padding: 10px 20px 30px 40px;` | 上・右・下・左（時計回り） |

```css
/* ❌ NG：バラバラに書く */
padding-top: 10px;
padding-bottom: 10px;
padding-left: 230px;
padding-right: 230px;

/* ⭕ OK：まとめて書く */
padding: 10px 230px;
```

### リンクの色は明示的に指定する

`<a>`タグはブラウザのデフォルトで青色＋下線になる。デザイン通りにするには、色を明示的に指定する。

```css
a {
  color: black;
  text-decoration: none;
}
```

### レスポンシブは変更箇所のみ記載する

メディアクエリ内には、変更するプロパティのみ書く。変更しないプロパティは削除する。

```css
/* PCスタイル */
.title {
  font-size: 1.5rem;
  color: #333;
  margin-top: 2.5rem;
}

/* ❌ NG：全部コピーして一部だけ変更 */
@media (max-width: 768px) {
  .title {
    font-size: 1.125rem;  /* ← 変更 */
    color: #333;          /* ← 変更なし（不要） */
    margin-top: 2.5rem;   /* ← 変更なし（不要） */
  }
}

/* ⭕ OK：変更箇所のみ */
@media (max-width: 768px) {
  .title {
    font-size: 1.125rem;
  }
}
```
