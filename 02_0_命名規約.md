# 命名規約（改訂版）

## 基本ルール

```
[何の]_[役割]
```

具体的に、何のセクションの何かがわかるようにする。

---

## タグ名とクラス名は別にする

HTMLタグと同じ名前をクラス名に使わない。

```html
<!-- ❌ NG：タグ名とクラス名が完全一致 -->
<header class="header">
<nav class="nav">
<footer class="footer">

<!-- ⭕ OK：接頭語で区別する -->
<header class="site_header">
<nav class="global_nav">
<footer class="site_footer">
```

**ルール：**
- `<header class="header">` → ❌ 完全一致はNG
- `<header class="site_header">` → ⭕ 接頭語があればOK
- `<header class="main_header">` → ⭕ 接頭語があればOK

---

## 接尾語一覧

### 構造系

| 役割 | 接尾語 | 使いどころ | 例 |
|------|--------|-----------|-----|
| ページ全体 | `wrapper` | body直下のコンテナ | `wrapper` |
| 幅制限・中央寄せ | `_container` | max-width指定する要素 | `about_container` |
| 内側の余白 | `_inner` | paddingで余白を作る要素 | `site_header_inner` |
| セクション | `_area` | 背景色・余白が必要な場合のみ | `hero_area`, `about_area` |

**`_area`の使用基準：**
- section自体に背景色や余白を当てる場合のみ使う
- 中の`_container`で十分な場合は不要

---

### Flex親（並べる系）

| 役割 | 接尾語 | 使いどころ | 例 |
|------|--------|-----------|-----|
| 同じ要素を並べる | `_list` | カード3枚、アイコン4つなど | `recipe_list`, `sns_list` |
| 異なる要素を並べる | `_row` | 画像+テキスト、ロゴ+ナビなど | `recipe_row`, `header_row` |

**判断方法：**
```
同じものを複数？ → _list
  例：カード、カード、カード

違うものを2つ以上？ → _row
  例：画像、テキスト情報
```

---

### Flex子（並べられる側）

| 役割 | 接尾語 | 判断基準 | 例 |
|------|--------|---------|-----|
| 単体要素 | `_item` | 背景色・枠線が不要 | `sns_item`, `nav_item` |
| カード状の要素 | `_card` | 背景色・枠線が必要 | `recipe_card`, `blog_card` |

**超シンプル判断法：**

```
質問：「この要素に背景色や枠線を付けたいか？」

YES → _card
NO  → _item
```

**具体例：**
```html
<!-- 背景色いらない → _item -->
<a class="sns_item">
  <img src="icon.svg">
  <span>Instagram</span>
</a>

<!-- 背景色ほしい → _card -->
<div class="recipe_card">
  <img src="food.jpg">
  <h3>レシピ名</h3>
  <p>説明文</p>
</div>
```

---

### 内容系

| 役割 | 接尾語 | 例 |
|------|--------|-----|
| 見出し | `_title` | `recipe_title` |
| 説明文 | `_description` | `recipe_description` |
| 画像 | `_image` | `recipe_image` |
| ボタン | `_button` | `contact_button` |
| リンク | `_link` | `footer_link` |
| 著作権 | `_copyright` | `site_footer_copyright` |
| 注釈 | `_note` | `price_note` |
| キャプション | `_caption` | `image_caption` |
| SP専用改行 | `sp_br` | `sp_br` |

### 固定の名前

| 役割 | クラス名 |
|------|----------|
| ページ全体 | `wrapper` |
| サイトヘッダー | `site_header` |
| サイトフッター | `site_footer` |
| グローバルナビ | `global_nav` |

## 「_text」は使わない

「text」は曖昧になりやすいので、具体的な名前をつける。

```
❌ intro_text       → 何のテキスト？
⭕ intro_description  → 説明文
⭕ intro_note         → 注釈
⭕ intro_caption      → キャプション
```

「_text」を使いそうになったら「これ本当は何？」と考える。

## Flexの使い分け（超重要）

### 結論（これだけ覚える）

| 親要素 | 子要素の背景色・枠線 | 子要素のflex | 使う接尾語 |
|--------|---------------------|-------------|-----------|
| **flex使う** | **あり** | 使う・使わない関係なし | `_card` |
| **flex使う** | **なし** | 使う・使わない関係なし | `_item` |
| flex使わない | - | - | `_container` / `_inner` |

**判断基準は2つだけ：**
1. ✅ 親がflexを使ってるか？
2. ✅ 子に背景色・枠線があるか？

**❌ 子自身がflexを使うかどうかは関係ない**

---

### 判断フローチャート

```
親要素がflexを使ってる？
 │
 ├─ NO → _container または _inner
 │
 └─ YES → 子要素に背景色・枠線がある？
     │
     ├─ YES → _card
     │
     └─ NO → _item（または固有名）
```

---

### パターン1: `_container`（flexなし）

**構造図:**
```
about_container
 ├─ タイトル
 ├─ 説明文
 └─ ボタン
```

**HTML:**
```html
<div class="about_container">
  <h2>タイトル</h2>
  <p>説明文</p>
</div>
```

**CSS:**
```css
.about_container {
  max-width: 1200px;  /* 幅制限のみ */
  margin: 0 auto;     /* flexは使わない */
}
```

**いつ使う：** 幅制限と中央寄せだけ

---

### パターン2: 親flex + 子flexなし + 背景色なし → `_item`

**構造図:**
```
sns_list ← 親がflex
 ├─ sns_item ← 子はflexなし + 背景色なし
 │   └─ Instagram画像
 ├─ sns_item
 │   └─ Twitter画像
 └─ sns_item
     └─ Facebook画像
```

**HTML:**
```html
<div class="sns_list">
  <a class="sns_item" href="#">
    <img src="instagram.svg">
  </a>
  <a class="sns_item" href="#">
    <img src="twitter.svg">
  </a>
</div>
```

**CSS:**
```css
.sns_list {
  display: flex;  /* ← 親がflex */
  gap: 10px;
}

.sns_item {
  /* flexは使わない */
  /* 背景色なし */
}
```

**いつ使う：** 同じ単体要素を並べる、背景色なし

---

### パターン3: 親flex + 子flex + 背景色あり → `_card`

**構造図:**
```
recipe_list ← 親がflex
 ├─ recipe_card ← 子もflex + 背景色あり
 │   ├─ recipe_card_image
 │   ├─ recipe_card_title
 │   └─ recipe_card_description
 └─ recipe_card ← 子もflex + 背景色あり
     ├─ recipe_card_image
     ├─ recipe_card_title
     └─ recipe_card_description
```

**HTML:**
```html
<div class="recipe_list">
  <div class="recipe_card">
    <img class="recipe_card_image" src="food1.jpg">
    <h3 class="recipe_card_title">レシピ1</h3>
    <p class="recipe_card_description">説明文</p>
  </div>
</div>
```

**CSS:**
```css
.recipe_list {
  display: flex;  /* ← 親がflex */
  gap: 20px;
}

.recipe_card {
  display: flex;          /* ← 子もflex */
  flex-direction: column;
  background-color: #fff; /* ← 背景色あり */
  border: 1px solid #ddd;
  padding: 20px;
}
```

**いつ使う：** カードを複数並べる、カードの中もflexで整列、背景色あり

---

### パターン4: 親flex + 子flexなし + 背景色あり → `_card`

**構造図:**
```
product_list ← 親がflex
 ├─ product_card ← 子はflexなし + 背景色あり
 │   ├─ 商品画像
 │   └─ 商品名
 └─ product_card ← 子はflexなし + 背景色あり
     ├─ 商品画像
     └─ 商品名
```

**HTML:**
```html
<div class="product_list">
  <div class="product_card">
    <img src="product1.jpg">
    <p>商品名1</p>
  </div>
</div>
```

**CSS:**
```css
.product_list {
  display: flex;  /* ← 親がflex */
  gap: 20px;
}

.product_card {
  /* flexは使わない（自然に縦並び） */
  background-color: #fff; /* ← 背景色あり */
  border: 1px solid #ddd;
  padding: 20px;
}
```

**いつ使う：** カードを並べる、子はflexを使わない、背景色あり

---

### パターン5: 親flex + 子flex + 背景色なし → `_item`（または固有名）

**構造図:**
```
header_row ← 親がflex
 ├─ site_logo ← 子もflex + 背景色なし
 │   └─ ロゴ画像
 └─ global_nav ← 子もflex + 背景色なし
     └─ global_nav_list
         ├─ リンク1
         └─ リンク2
```

**HTML:**
```html
<div class="header_row">
  <div class="site_logo">
    <img src="logo.svg">
  </div>
  <nav class="global_nav">
    <ul class="global_nav_list">
      <li><a href="#">HOME</a></li>
    </ul>
  </nav>
</div>
```

**CSS:**
```css
.header_row {
  display: flex;  /* ← 親がflex */
  justify-content: space-between;
}

.site_logo {
  display: flex;  /* ← 子もflex */
  /* 背景色なし */
}

.global_nav_list {
  display: flex;  /* ← 子もflex */
  gap: 20px;
  /* 背景色なし */
}
```

**いつ使う：** 異なる要素を並べる、子もflexを使う、背景色なし  
**補足：** この例では固有名（`site_logo`, `global_nav`）を使用

---

### よくある間違い

#### ❌ 間違い1: 「flexが2回使われてるから`_card`」

```html
<div class="header_row">  <!-- flex -->
  <nav class="global_nav">  <!-- flex + 背景色なし -->
    <!-- ナビの中身 -->
  </nav>
</div>
```

**正しい判断：**
- 親がflex ✅
- 背景色なし ❌
- **→ `_item`または固有名（この場合は`global_nav`）**

---

#### ❌ 間違い2: 「子がflexを使ってないから`_item`」

```html
<div class="blog_list">  <!-- flex -->
  <div class="blog_card">  <!-- flexなし + 背景色あり -->
    <img src="blog.jpg">
    <h3>ブログタイトル</h3>
  </div>
</div>
```

**正しい判断：**
- 親がflex ✅
- 背景色あり ✅
- **→ `_card`**（子がflexを使ってなくても関係ない）

## よくあるパターン

### ナビゲーション

```html
<nav class="global_nav">
  <ul class="global_nav_list">
    <li><a class="global_nav_link">HOME</a></li>
    <li><a class="global_nav_link">ABOUT</a></li>
    <li><a class="global_nav_link">CONTACT</a></li>
  </ul>
</nav>
```

| クラス名 | 役割 |
|----------|------|
| `global_nav` | navタグ自体 |
| `global_nav_list` | flexの親 |
| `global_nav_link` | リンク（スタイル対象） |
| `<li>` | クラス不要（スタイルしない） |

---

### ヘッダー（背景全幅、中身は幅制限）

```html
<header class="site_header">
  <div class="site_header_inner">
    <h1 class="site_logo">ロゴ</h1>
    <nav class="global_nav">...</nav>
  </div>
</header>
```

```css
.site_header {
  background-color: #fff; /* 背景は画面全体 */
}

.site_header_inner {
  max-width: 1200px;  /* 中身だけ幅制限 */
  margin: 0 auto;
  padding: 0 20px;    /* 左右余白 */
}
```

**`_container`と`_inner`はflexを使わない**
- 目的：幅制限と余白調整のみ
- flexが必要な場合は、中に`_list`や`_row`を入れる

---

### セクション（基本パターン）

```html
<section class="about_area">
  <div class="about_container">
    <h2 class="about_title">タイトル</h2>
    <p class="about_description">説明文</p>
  </div>
</section>
```

```css
.about_area {
  background-color: #f5f5f5; /* セクション全体の背景 */
  padding: 80px 0;
}

.about_container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 20px;
}
```

**重要：**
- **sectionタグ = `_area`**
- **`_area`は背景色が必要な場合のみ使う**
- **`_container`はflexを使わない**（幅制限のみ）

---

### カードリスト（flexが2回続く場合）

```html
<div class="recipe_list">        <!-- ①flexの親 -->
  <div class="recipe_card">      <!-- ②flexの子（背景色あり） -->
    <img class="recipe_card_image" src="food1.jpg">
    <h3 class="recipe_card_title">レシピ1</h3>
    <p class="recipe_card_description">説明文</p>
  </div>

  <div class="recipe_card">
    <img class="recipe_card_image" src="food2.jpg">
    <h3 class="recipe_card_title">レシピ2</h3>
    <p class="recipe_card_description">説明文</p>
  </div>
</div>
```

```css
/* ①flexでカードを横並び */
.recipe_list {
  display: flex;
  gap: 20px;
}

/* ②カードの中もflexで縦並び */
.recipe_card {
  display: flex;
  flex-direction: column;
  background-color: #fff;  /* 背景色あり */
  border: 1px solid #ddd;  /* 枠線あり */
}
```

**いつ使う？**
- `_list` → flexで要素を並べる（横並び・縦並び）
- `_card` → その中の1つ1つで、**さらにflexを使う** + **背景色・枠線あり**

**判断基準：**
1. 親がflex → `_list`
2. 子もflex + 背景色・枠線あり → `_card`
3. 子はflexじゃない or 背景色なし → `_item`

---

### SNSアイコン（flexが1回だけの場合）

```html
<div class="sns_list">          <!-- ①flexの親 -->
  <a class="sns_item" href="#">  <!-- ②flexの子（背景色なし） -->
    <img src="instagram.svg">
  </a>
  <a class="sns_item" href="#">
    <img src="twitter.svg">
  </a>
  <a class="sns_item" href="#">
    <img src="facebook.svg">
  </a>
</div>
```

```css
/* flexでアイコンを横並び */
.sns_list {
  display: flex;
  gap: 10px;
}

/* 子要素はflexじゃない */
.sns_item {
  /* 背景色なし、枠線なし */
}
```

**いつ使う？**
- `_list` → flexで要素を並べる
- `_item` → その中の1つ1つで、**flexは使わない** or **背景色・枠線なし**

---

### 画像+テキスト（flexで異なる要素を並べる）

```html
<div class="intro_row">             <!-- ①flexの親 -->
  <img class="intro_image" src="about.jpg">     <!-- 画像 -->
  <div class="intro_text_area">                 <!-- テキストエリア -->
    <h2 class="intro_title">タイトル</h2>
    <p class="intro_description">説明文</p>
  </div>
</div>
```

```css
/* flexで画像とテキストを横並び */
.intro_row {
  display: flex;
  gap: 40px;
  align-items: center;
}
```

**`_row`を使う理由：**
- **flexで異なる種類の要素**を並べる（画像 + テキストエリア、ロゴ + ナビなど）
- `_list`との違い：同じものを複数 = `_list`、違うものを並べる = `_row`

## 判断フローチャート

```
クラス名を付けたい要素がある
    │
    ├─ ページ全体？ → wrapper
    │
    ├─ セマンティックタグ（header/nav/footer）？
    │   → site_header / global_nav / site_footer
    │
    ├─ 幅制限・中央寄せしたい？ → [内容]_container
    │
    ├─ 内側に余白を作りたい？ → [内容]_inner
    │
    ├─ sectionタグで背景色が必要？ → [内容]_area
    │   （背景色不要なら省略してOK）
    │
    ├─ flexで子要素を並べる親？
    │   │
    │   ├─ 同じ種類のものを複数？ → [内容]_list
    │   │   例：カード3枚、アイコン4つ
    │   │
    │   └─ 違う種類のものを2つ以上？ → [内容]_row
    │       例：画像とテキスト情報、ロゴとナビ
    │
    ├─ 並んでる中の1つ1つ？
    │   │
    │   ├─ 背景色・枠線が必要？ → [内容]_card
    │   │
    │   └─ 背景色・枠線が不要？ → [内容]_item
    │
    └─ その他の要素？ → [どこの]_[役割]
        │
        ├─ 見出し → _title
        ├─ 説明文 → _description
        ├─ 画像 → _image
        ├─ ボタン → _button
        ├─ リンク → _link
        └─ 著作権 → _copyright
```

## チェックリスト

命名時に確認すること：

- [ ] タグ名とクラス名が完全一致していないか？
- [ ] 接尾語が正しいか？（_area, _container, _list, _row, _item, _card 等）
- [ ] `_text` を使っていないか？（_description, _note, _caption を使う）
- [ ] [何の]_[役割] の形式になっているか？
- [ ] スペルミスがないか？
- [ ] すべて小文字か？（大文字小文字が混在していないか）
- [ ] 背景色・枠線が必要なら `_card`、不要なら `_item` を使っているか？

## 優先度

最初から全部使う必要はない。まずこれだけ覚える：

### 最優先（必ず使う）

- `wrapper`
- `site_header`, `site_footer`, `global_nav`
- `_list`, `_item`
- `_title`, `_description`, `_image`, `_button`, `_link`

### 次に覚える

- `_container`
- `_row`
- `_card`

### 後でいい

- `_inner`（必要になったら追加）
- `_area`（背景色が必要な場合のみ）

## まとめ

1. **迷ったら「背景色付けるか？」で判断**
2. **`_text`は使わない**（具体的な名前を付ける）
3. **最初は基本だけ使って、必要に応じて追加する**
