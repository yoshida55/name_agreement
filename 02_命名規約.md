命名規約（改訂版）
基本ルール
[何の]_[役割]
具体的に、何のセクションの何かがわかるようにする。

タグ名とクラス名は別にする
HTMLタグと同じ名前をクラス名に使わない。
html<!-- ❌ NG：タグ名とクラス名が完全一致 -->
<header class="header">
<nav class="nav">
<footer class="footer">

<!-- ⭕ OK：接頭語で区別する -->
<header class="site_header">
<nav class="global_nav">
<footer class="site_footer">
ルール：

<header class="header"> → ❌ 完全一致はNG
<header class="site_header"> → ⭕ 接頭語があればOK
<header class="main_header"> → ⭕ 接頭語があればOK


接尾語一覧
構造系
役割接尾語使いどころ例ページ全体wrapperbody直下のコンテナwrapper幅制限・中央寄せ_containermax-width指定する要素about_container内側の余白_innerpaddingで余白を作る要素site_header_innerセクション_area背景色・余白が必要な場合のみhero_area, about_area
_areaの使用基準：

section自体に背景色や余白を当てる場合のみ使う
中の_containerで十分な場合は不要


Flex親（並べる系）
役割接尾語使いどころ例同じ要素を並べる_listカード3枚、アイコン4つなどrecipe_list, sns_list異なる要素を並べる_row画像+テキスト、ロゴ+ナビなどrecipe_row, header_row
判断方法：
同じものを複数？ → _list
  例：カード、カード、カード

違うものを2つ以上？ → _row
  例：画像、テキスト情報

Flex子（並べられる側）
役割接尾語判断基準例単体要素_item背景色・枠線が不要sns_item, nav_itemカード状の要素_card背景色・枠線が必要recipe_card, blog_card
超シンプル判断法：
質問：「この要素に背景色や枠線を付けたいか？」

YES → _card
NO  → _item
具体例：
html<!-- 背景色いらない → _item -->
<a class="sns_item">
  <img src="icon.svg">
  <span>Instagram</span>
</a>

<!-- 背景色ほしい → _card -->
<div class="recipe_card">
  <img src="food.jpg">
  <h3>レシピ名</h3>
  <p>説明文</p>
</div>

内容系
役割接尾語例見出し_titlerecipe_title説明文_descriptionrecipe_description画像_imagerecipe_imageボタン_buttoncontact_buttonリンク_linkfooter_link著作権_copyrightsite_footer_copyright注釈_noteprice_noteキャプション_captionimage_captionSP専用改行sp_brsp_br

固定の名前
役割クラス名ページ全体wrapperサイトヘッダーsite_headerサイトフッターsite_footerグローバルナビglobal_nav

「_text」は使わない
「text」は曖昧になりやすいので、具体的な名前をつける。
❌ intro_text       → 何のテキスト？
⭕ intro_description  → 説明文
⭕ intro_note         → 注釈
⭕ intro_caption      → キャプション
「_text」を使いそうになったら「これ本当は何？」と考える。

判断フローチャート
クラス名を付けたい要素がある
    │
    ├─ ページ全体？ → wrapper
    │
    ├─ セマンティックタグ（header/nav/footer）？
    │   → site_header / global_nav / site_footer
    │
    ├─ 幅制限・中央寄せしたい？ → [内容]_container
    │
    ├─ 内側に余白を作りたい？ → [内容]_inner
    │
    ├─ sectionタグで背景色が必要？ → [内容]_area
    │   （背景色不要なら省略してOK）
    │
    ├─ flexで子要素を並べる親？
    │   │
    │   ├─ 同じ種類のものを複数？ → [内容]_list
    │   │   例：カード3枚、アイコン4つ
    │   │
    │   └─ 違う種類のものを2つ以上？ → [内容]_row
    │       例：画像とテキスト情報、ロゴとナビ
    │
    ├─ 並んでる中の1つ1つ？
    │   │
    │   ├─ 背景色・枠線が必要？ → [内容]_card
    │   │
    │   └─ 背景色・枠線が不要？ → [内容]_item
    │
    └─ その他の要素？ → [どこの]_[役割]
        │
        ├─ 見出し → _title
        ├─ 説明文 → _description
        ├─ 画像 → _image
        ├─ ボタン → _button
        ├─ リンク → _link
        └─ 著作権 → _copyright

チェックリスト
命名時に確認すること：

 タグ名とクラス名が完全一致していないか？
 接尾語が正しいか？（_area, _container, _list, _row, _item, _card 等）
 _text を使っていないか？（_description, _note, _caption を使う）
 [何の]_[役割] の形式になっているか？
 スペルミスがないか？
 すべて小文字か？（大文字小文字が混在していないか）
 背景色・枠線が必要なら _card、不要なら _item を使っているか？


優先度
最初から全部使う必要はない。まずこれだけ覚える：
最優先（必ず使う）

wrapper
site_header, site_footer, global_nav
_list, _item
_title, _description, _image, _button, _link

次に覚える

_container
_row
_card

後でいい

_inner（必要になったら追加）
_area（背景色が必要な場合のみ）
