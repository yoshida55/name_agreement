# コーディング規約

## フォーマット

- インデントはスペース 2 個
- コードは全て小文字
- 長いテキストはコード内で改行せず、VS Code の折り返し表示を使う

## 外部リンク

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer"></a>
```

※ `rel="noopener noreferrer"` はセキュリティ対策（リンク先から元ページを操作されるのを防ぐ）

## 画像の alt

「〜です」のような文章は書かない。名詞または短い説明で書く。

```
⭕ OK（名詞・短い説明）
  alt="自転車"
  alt="My Workのロゴ"
  alt="宮地翔のプロフィール写真"

❌ NG（文章）
  alt="これは自転車の写真です"
  alt="こちらが私たちのロゴになります"
```

## セレクタ

- 深掘りは 1 段階まで（子孫セレクタは 1 回まで）
- タグを直接指定しない
- `position: absolute` は最終手段

```css
/* ⭕ OK: 1段階（親 → 子） */
.header .logo {
}
.nav_list .nav_link {
}

/* ❌ NG: 2段階以上（親 → 子 → 孫） */
.header .nav .nav_link {
}
.recipe_list .recipe_card .recipe_card_image {
}
```

## 【詳細解説】わかりにくいルール 3 つ

### 1. セクションの役割はコメントで残す

HTML を後から見たとき「ここ何のセクションだっけ？」とならないように、
コメントで目印を残す。

```html
<!-- ❌ NG：コメントなし。後から見ると何が何だかわからない -->
<section>
  <h1>Recipe Diary</h1>
  <p>日々の料理レシピをまとめています。</p>
</section>
```

```html
<!-- ⭕ OK：セクションごとに役割を書いておく -->

<!-- ===============================================
 サイト紹介
=============================================== -->
<section class="intro">
  <h1>Recipe Diary</h1>
  <p>日々の料理レシピをまとめています。</p>
</section>
```

**なぜ？**
→ 100 行、200 行と増えたとき、コメントがないとスクロールして探し回ることになる。

---

### 2. リスト・テーブルは改行してインデントする

入れ子構造は「親子関係」が見えるように書く。

```html
<!-- ⭕ OK：改行とインデントで親子関係が見える -->
<ul>
  <li>Instagram</li>
  <li>Twitter</li>
  <li>Facebook</li>
</ul>
```

**なぜ？**
→ 後から「li を 1 個追加したい」となったとき、どこに書けばいいか一目でわかる。

---

### 3. 一連の文章を 1 行ずつ p タグで区切らない

「見た目で改行したいから」という理由で p タグを分けない。

```html
<!-- ❌ NG：見た目の改行のためにpタグを分けている -->
<p>日々の料理レシピをまとめています。</p>
<p>和食や洋食、中華、お菓子まで</p>
<p>いろいろな料理レシピをアップしています。</p>
```

```html
<!-- ⭕ OK：ひとつの段落はひとつのpタグ -->
<p>
  日々の料理レシピをまとめています。
  和食や洋食、中華、お菓子までいろいろな料理レシピをアップしています。
</p>
```

行間を空けたい場合は CSS で調整する：

```css
p {
  line-height: 1.8;
}
```

**なぜ？**
→ p タグは「段落（意味のまとまり）」を表す。
→ 見た目のために HTML をいじると、後でデザイン変更したとき全部直す羽目になる。
