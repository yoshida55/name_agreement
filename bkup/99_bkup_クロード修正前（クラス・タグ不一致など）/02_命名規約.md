# 命名規則まとめ

## クラス名の形式

```
[何の]_[役割]
```

具体的に、何のセクションの何かがわかるようにする。

---

## 役割の接尾語一覧

| 役割                            | 接尾語         | 例                         |
| ------------------------------- | -------------- | -------------------------- |
| section タグ                    | `_area`        | `recipe_image_area`        |
| flex の親（同じ要素を並べる）   | `_list`        | `recipe_image_list`        |
| flex の親（異なる要素を並べる） | `_row`         | `recipe_detail_row`        |
| list の中（単体）               | `_item`        | `sns_item`                 |
| list の中（入れ子 flex）        | `_card`        | `recipe_card`              |
| タイトル                        | `_title`       | `recipe_diary_title`       |
| 説明文                          | `_description` | `recipe_diary_description` |
| 画像                            | `_image`       | `recipe_image`             |
| ボタン                          | `_button`      | `recipe_list_button`       |
| リンク                          | `_link`        | `footer_sns_link`          |
| 著作権                          | `_copyright`   | `footer_copyright`         |
| 注釈                            | `_note`        | `price_note`               |
| キャプション                    | `_caption`     | `image_caption`            |
| SP 専用改行                     | `sp_br`        | `sp_br`                    |
| ページ全体                      | `wrapper`      | `wrapper`                  |
| ヘッダー                        | `header`       | `header`                   |
| フッター                        | `footer`       | `footer`                   |
| ナビ                            | `nav`          | `nav`                      |

---

## 「\_text」は使わない

「text」は曖昧になりやすいので、具体的な名前をつける。

```
❌ intro_text    → 何のテキスト？
⭕ intro_description  → 説明文
⭕ intro_note         → 注釈
⭕ intro_caption      → キャプション
```

「\_text」を使いそうになったら「これ本当は何？」と考える。

## ナビゲーションの構造

```html
<nav class="nav">
  <ul class="nav_list">
    <li><a class="nav_link">HOME</a></li>
    <li><a class="nav_link">ABOUT</a></li>
    <li><a class="nav_link">CONTACT</a></li>
  </ul>
</nav>
```

- `nav` → nav タグ自体
- `nav_list` → flex の親
- `nav_link` → 中のリンク
- `<li>` → スタイリングしないのでクラス不要

※ CSS でスタイルを当てない要素にはクラスを付けなくて OK

---

## flex の使い分け：\_list と \_row

### `_list`（同じ要素を複数並べる）

```html
<div class="sns_list">
  <a class="sns_item">Instagram</a>
  <a class="sns_item">Twitter</a>
  <a class="sns_item">Facebook</a>
</div>
```

### `_row`（異なる要素を横に並べる）

```html
<div class="recipe_detail_row">
  <img class="recipe_detail_image" />
  <div class="recipe_detail_info">
    <h1>タイトル</h1>
    <p>説明文</p>
  </div>
</div>
```

| 状況                   | 接尾語  | 使いどころ                         |
| ---------------------- | ------- | ---------------------------------- |
| 同じ要素を複数並べる   | `_list` | カード、SNS アイコン、ナビリンク等 |
| 異なる要素を横に並べる | `_row`  | 画像＋テキスト、ロゴ＋ナビ等       |

---

## flex の入れ子：\_item と \_card の使い分け

**1 段階の flex（中身が単体）**

```html
<div class="sns_list">
  <a class="sns_item">Instagram</a>
  <a class="sns_item">Twitter</a>
  <a class="sns_item">Facebook</a>
</div>
```

**2 段階の flex（中がまた flex になる）**

```html
<div class="recipe_list">
  <div class="recipe_card">
    <img class="recipe_card_image" />
    <p class="recipe_card_title"></p>
  </div>

  <div class="recipe_card">
    <img class="recipe_card_image" />
    <p class="recipe_card_title"></p>
  </div>
</div>
```

| 構造             | 名前                     |
| ---------------- | ------------------------ |
| 1 段階           | `_list` → `_item`        |
| 2 段階（入れ子） | `_list` → `_card` → 中身 |

---

## 判断フローチャート

```
クラス名を付けたい要素がある
    │
    ├─ sectionタグ？ → [内容]_area
    │
    ├─ flexで子要素を並べる親？
    │   │
    │   ├─ 同じ種類のものを複数？ → [内容]_list
    │   │   例：カード3枚、アイコン4つ
    │   │
    │   └─ 違う種類のものを2つ以上？ → [内容]_row
    │       例：画像とテキスト情報、ロゴとナビ
    │
    ├─ 並んでる中の1つ1つ？ → [内容]_image, [内容]_link など
    │
    └─ その他の要素？ → [どこの]_[役割]
        │
        ├─ 見出し → _title
        ├─ 説明文 → _description
        ├─ 画像 → _image
        ├─ ボタン → _button
        ├─ リンク → _link
        └─ 著作権 → _copyright
```
